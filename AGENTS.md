# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

## Commands

- `bun install` — install dependencies
- `bun dev` — dev server on `:3000`
- `bun run lint` — ESLint (flat config, `eslint.config.mjs`)
- `bun run build` — production build; also catches type errors
- No test framework is configured
- No dedicated typecheck script; run `bunx tsc --noEmit` if needed

## Stack

- **Next.js 16** App Router, **React 19**, **Tailwind CSS v4**, **shadcn/ui v4** (style: `base-nova`)
- `nuqs` for URL search-param state — always wrap state in `NuqsAdapter` (already in root layout)
- `recharts` for charts, `lucide-react` for icons
- Path alias: `@/*` → `./src/*`

## Architecture

Single-page app with one API route:

- `src/app/page.tsx` — main client component (`"use client"`); fetches models from `/api/models` on mount
- `src/app/api/models/route.ts` — proxies `https://api.kilo.ai/api/gateway/models` with 1-hour server cache (`revalidate: 3600`)
- `src/app/models/[id]/page.tsx` — model detail page
- `src/lib/types.ts` — `Model`, `ModelsResponse`, `ViewMode`, `ColorTheme` types
- `src/hooks/` — `use-models.ts`, `use-model-filters.ts`, `use-favorites.ts`
- `src/components/` — feature components; `ui/` subfolder is auto-generated by shadcn

### Data flow

The Kilo API is called server-side only. The client fetches from the local `/api/models` route. No `.env` files are needed — the API URL is hardcoded in the route handler.

## Conventions

- `cn()` utility from `@/lib/utils` (clsx + tailwind-merge) for class composition
- Multiple color themes are defined as CSS classes in `src/app/globals.css` (e.g. `.catppuccin`, `.nord`, `.dracula`). New themes go there, not in Tailwind config.
- Add shadcn components with `bunx shadcn@latest add <component>` — they land in `src/components/ui/`
- ESLint uses the flat config format (`eslint.config.mjs`), not `.eslintrc`
